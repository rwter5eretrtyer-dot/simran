# 🏝 Island Project (Отчёт по тестированию)

## 📌 Описание проекта

Данный проект представляет собой учебную модель «Остров», в рамках которой реализован простой объект **Animal** с базовой логикой поведения.

Основная идея проекта — не создание сложной системы, а демонстрация навыков тестирования программного обеспечения с использованием различных технологий.

В проекте реализовано тестирование на трёх платформах:
- Java (JUnit 5)
- .NET (xUnit и NUnit)
- JavaScript (Jest)

---

## 🎯 Цель работы

Целью проекта является:
- разработка простой логики, пригодной для тестирования;
- покрытие кода unit и функциональными тестами;
- применение техник тестирования;
- оформление результатов тестирования в виде документации.

---

## 🧩 Описание логики

В проекте реализован класс **Animal**, который содержит:
- поле `energy` (энергия объекта);
- метод `move()`, уменьшающий энергию при движении.

Логика простая и прозрачная, что позволяет сосредоточиться на тестировании и корректной проверке поведения системы.

---

## 🧪 Тестирование

### 🔹 Java (JUnit)

Реализовано:
- 10 unit-тестов
- 2 функциональных теста

**Что проверяется:**
- уменьшение энергии после вызова метода `move()`;
- поведение при энергии = 0 (граничное значение);
- работа на стандартных значениях;
- поведение при отрицательных значениях;
- корректность при нескольких последовательных вызовах метода.

---

### 🔹 .NET (xUnit и NUnit)

Реализовано:
- 5 unit-тестов на xUnit
- 5 unit-тестов на NUnit
- 1 функциональный тест

**Что проверяется:**
- корректность вычислений;
- стабильность работы метода;
- выполнение типового сценария использования.

---

### 🔹 JavaScript (Jest)

Реализовано:
- 10 unit-тестов
- 2 функциональных теста

**Что проверяется:**
- корректность функции;
- работа на различных входных данных;
- выполнение функциональных сценариев.

---

## 🧠 Использованные техники тестирования

В рамках проекта применены следующие техники:

### ✔ Граничные значения
Проверка поведения системы на крайних значениях (например, энергия = 0).

### ✔ Эквивалентное разбиение
Разделение входных данных на группы:
- положительные значения
- ноль
- отрицательные значения

### ✔ Негативные сценарии
Проверка поведения системы при некорректных или нестандартных входных данных.
